How to Use These Skills

Type the skill name as a slash command in Claude Code — e.g. /vitality-daily-brief — then paste your content. Each skill knows what to ask for if context is missing.

Skills are stored at ~/MySkills/vitality-*/SKILL.md and are personal to your Claude Code installation. Start with /vitality-explore to get a personalized onboarding prompt for your exact workflow.

Skills work best when you paste real content — Slack excerpts, call notes, survey data, tracker exports — directly into the prompt. The more context, the fewer revision cycles.

Log Activity to CRM
/vitality-log-crm
Converts rough post-call notes into a structured, Salesforce-ready activity log. Handles batch mode — paste notes from multiple calls and it processes each one.
When to useAfter any customer call, onboarding session, check-in, or enrollment follow-up. Say "log this call" or "draft CRM entry".
EXAMPLE PROMPT
I just finished a 30-min onboarding call with Acme Corp. Notes:
- Spoke with Jane (IT lead)
- Device arrived, not set up yet
- Privacy concern: who sees data?
- Next: send privacy FAQ by Friday
Log this to Salesforce.
Adapt Content Across Platforms
/vitality-adapt
Takes one Vitality message and outputs Slack post + manager email + intranet article + slide talking points — all at once, without rewriting from scratch.
When to useWhen the same update needs to go to multiple channels. Say "adapt this for Slack and email" or "repurpose this content".
EXAMPLE PROMPT
Here's our Q2 enrollment update: we hit 42% enrollment,
up from 30%. Next milestone is 60% by June 30.
Adapt into: Slack post, manager email, 3 slide talking points.
Match the tone of this example email: [paste]
Analyze Patterns in User Feedback
/vitality-analyze-feedback
Extracts themes, sentiment breakdown, frequency counts, and quick-win recommendations from surveys, support emails, and Slack feedback — all at once.
When to useAfter survey results land or when you've collected a batch of user feedback. Say "analyze this feedback" or "what are users saying".
EXAMPLE PROMPT
Here are 3 sources of Vitality feedback from the past 60 days:
Survey responses: [paste CSV or text]
Support emails: [paste summaries]
Slack excerpts: [paste]
Find top themes, sentiment, and 3 quick wins.
Generate Project Status Reports
/vitality-status-report
Consolidates Slack updates, emails, and tracker notes into a structured weekly program status report. Flags blockers, reconciles conflicting numbers, and produces an executive summary.
When to useWeekly or monthly reporting to leadership. Say "generate status report" or "weekly Vitality update".
EXAMPLE PROMPT
Here are updates from this week [paste Slack, email, tracker].
Generate a Vitality status report:
- Enrollment vs. target
- Onboarding cohort health
- Blockers >5 days old
- 3-sentence executive summary
Surface Themes from All Feedback Channels
/vitality-surface-themes
Cross-references ALL feedback sources and surfaces only validated themes — those appearing in 2+ channels. Separates real signal from one-off noise with a ranked priority list.
When to useQuarterly synthesis to confirm what's genuinely widespread. Say "cross-channel feedback analysis" or "what's validated signal".
EXAMPLE PROMPT
Synthesize Vitality feedback from Q1 across:
- Survey responses [paste]
- #vitality-questions Slack [paste]
- CS meeting notes [paste]
Show only themes appearing in 2+ sources, ranked by priority.
Source Insights to Build a Deck
/vitality-build-deck
Synthesizes enrollment data, wins, feedback themes, and blockers from multiple sources into a slide-by-slide outline with talking points and a 2-paragraph executive narrative.
When to useQBRs, stakeholder presentations, enrollment milestone decks. Say "build a deck" or "create a QBR presentation".
EXAMPLE PROMPT
Build a Vitality Q2 review deck. Sources:
- Enrollment metrics: 42%, target 60% by June [paste data]
- Wins from Slack [paste]
- Feedback themes [paste]
- Blockers [paste]
7 slides. Audience: leadership team.
Build a Daily Briefing
/vitality-daily-brief
Builds a structured morning briefing from pasted Slack messages, email updates, and tracker notes. Surfaces urgent items, at-risk customers, overdue follow-ups, and one strategic focus.
When to useStart of every workday. Say "daily brief", "morning briefing", or "what do I focus on today".
EXAMPLE PROMPT
Here are my Slack messages and emails from this morning [paste].
Build my daily Vitality briefing:
- Urgent actions
- At-risk customers
- Overdue follow-ups
- Today's strategic focus (15 min)
Package Brand Guidelines
/vitality-brand
Reviews any content against Geotab Vitality brand standards (color, tone, language rules) and delivers an on-brand rewrite. Also generates a reusable brand SKILL.md for future sessions.
When to useBefore sending any customer or internal communication. Say "make this on-brand", "apply brand guidelines", or "review this email".
EXAMPLE PROMPT
Review this onboarding email for brand compliance [paste].
Flag anything off-brand and deliver a corrected version.
─────────────────────
OR: Generate a brand skill file I can reuse in future sessions.
Explore What Claude Can Do
/vitality-explore
Maps your current tasks to Claude capabilities and returns 5 personalized, ready-to-run prompts with time-saved estimates. Ideal for onboarding new team members.
When to useFirst session with a new team member, or when you want to find underused capabilities. Say "what can Claude do for me" or "show me how to use this".
EXAMPLE PROMPT
I'm a Vitality CS Coordinator. My day includes:
- Responding to internal user questions
- Creating onboarding materials
- Tracking enrollment metrics
- Running engagement campaigns
Show me 5 ways Claude saves me time with ready-to-use prompts.
Create a Process Flowchart
/vitality-flowchart
Converts a written procedure, playbook, or escalation path into an ASCII decision tree + Mermaid.js code block (embeddable in Confluence or Notion), with role labels and exception paths.
When to useDocumenting onboarding procedures, troubleshooting guides, escalation paths. Say "create a flowchart", "visualize this process", or "make a decision tree".
EXAMPLE PROMPT
Convert this onboarding SOP [paste] into a decision-tree flowchart.
Show:
- Each step with owner (CS / Customer / IT)
- Decision points (device activated? Y/N)
- Exception paths
Also export Mermaid.js for Confluence.
Create a Company Newsletter
/vitality-newsletter
Synthesizes enrollment milestones, program wins, user spotlights, upcoming events, and FAQs into a polished, 500-word internal newsletter — scannable on mobile, Geotab-branded.
When to useMonthly program newsletter for internal stakeholders. Say "create newsletter", "write the Vitality newsletter", or "monthly digest".
EXAMPLE PROMPT
Create the April Vitality newsletter using:
- Enrollment: 42% enrolled [paste metrics]
- Wins from this month [paste]
- Upcoming: Q&A session April 15
- User spotlight: [paste notes]
- Top 2 FAQs this month [paste]
